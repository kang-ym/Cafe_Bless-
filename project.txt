커피 주문 사이트 제작 계획안

1. 제작 목표

- 모바일 중심의 커피 주문 웹사이트
- 고객과 관리자로 나뉜 시스템
- 간단한 주문 및 관리 기능 구현

2. 사용자 구분

(1) 고객 기능

- 커피 메뉴와 정보 확인 가능
- 커피 주문 (종류, 개수, hot/ice 선택, 주문자 이름 입력)
- 본인 이름은 설정 창이 따로 있어서 설정할 수 있고 주문시 자동 정보가 갈 수 있게 
- 만약 다른 사람 커피도 주문도 할 수도 있기에 따로 이름을 적을 수 있는 칸도 필요
- 간단한 주문서 작성 및 주문 버튼 클릭
- 주문이 끝나면 커피 준비중 화면 
- 커피가 만들어지만 커피 완료 화면 뜨고 클릭하면 처음화면으로 이동

(2) 관리자 기능

- 하루 동안의 고객 주문 현황 확인
- 주문 정보 (커피 종류, 개수, hot/ice, 주문자 이름) 보기
- 배달 완료 여부 체크 및 관리



3. 웹사이트 구성 요소

(1) 메인 페이지 (고객용)

- 커피 메뉴 표시 (아메리카노, 라떼, 바딜라라때, 카라멜라떼, 가격)
- 주문 버튼 및 폼 (이름, 수량, hot/ice)
- 커피 준비중 화면
- 커피 완성됫어요 화면 (touch)

(2) 관리자 페이지

- 로그인 기능 (간단한 ID/PW 인증)
- 주문 내역 리스트
- 배달 상태 체크박스

4. 개발에 사용할 기술

- HTML: 웹사이트 기본 구조
- CSS: 모바일 중심 레이아웃 및 스타일
- JavaScript, jQuery: 주문 폼 유효성 검사 및 동적인 기능 구현
- 향후 확장 및 유지보수를 고려하여 WordPress 활용 가능성 있음

5. 단계별 개발 계획

- 1단계: HTML/CSS로 기본 모바일 화면 구성
- 2단계: JavaScript, jQuery를 활용한 주문 기능 구현
- 3단계: 관리자 페이지 제작 및 주문 관리 기능 구현
- 4단계: 간단한 로그인 기능 추가 (관리자 페이지 보안)
- 5단계: 최종 테스트 및 수정, 반응형 디자인 최적화

6. 예상 난이도 및 학습 필요 기술

- 난이도: 중급
- 학습 필요 사항: JavaScript/JQuery를 이용한 데이터 처리, 간단한 인증 시스템

위 계획을 바탕으로 웹사이트 제작을 진행해 보겠습니다.
